# Plan: KidGameMaker Feature Suggestions — Massive Research & Writing Pipeline

## Objective
Read all project docs thoroughly, then research AAA studios (Nintendo, Konami, Capcom, Square/Enix, Sega, From Software) and indie studios for feature inspiration. Produce a comprehensive, well-organized feature suggestion document covering hundreds of new features and improvements, all designed so a 5-year-old can use them, with heavy LLM automation behind the scenes.

## Stage 1 — Parallel Deep Research (6+ agents)
Deploy research agents studying different studios/perspectives in parallel:

- **Agent 1: Nintendo Researcher** — Study Nintendo's kid-friendly design philosophy, Mario Maker 1/2 features, Mario Wonder, Zelda BotW/TotK, Kirby, Splatoon, Animal Crossing. Focus on creative tools, accessibility, emergent gameplay, level design features.
- **Agent 2: Konami/Capcom Researcher** — Study Castlevania, Mega Man, Monster Hunter, Street Fighter, Okami. Focus on progression systems, boss fights, weapon/item systems, combo mechanics.
- **Agent 3: Square Enix + Sega Researcher** — Study Final Fantasy, Kingdom Hearts, Sonic Mania, NiGHTS, Yuji Naka games. Focus on RPG elements, transformation systems, speed mechanics, narrative integration.
- **Agent 4: FromSoftware + Hardcore Indies Researcher** — Study Dark Souls, Elden Ring, Hollow Knight, Celeste, Shovel Knight, Ori. Focus on world design, tight controls, risk/reward, environmental storytelling.
- **Agent 5: Modern Indie Kids' Games Researcher** — Study Dreams (Media Molecule), Game Builder Garage, Super Mario Maker 2, Bloxels, Piskel, Scratch Jr, Kodu Game Lab, Tynker. Focus on how other kid-friendly game makers work, UI patterns, educational game design.
- **Agent 6: Modern Game UX & AI Researcher** — Study AI-assisted creation tools (Roblox AI, Minecraft Copilot, Midjourney, DALL-E integration), DreamScreen, auto-difficulty, procedural generation, modern accessibility features in games.

## Stage 2 — Feature Compilation & Writing
- Gather all research outputs
- Organize into logical categories (World Building, Characters, Mechanics, AI Automation, etc.)
- Write comprehensive feature suggestions with studio-inspiration attribution
- Ensure every feature has a "5-year-old UX" section explaining how it stays simple
- Use the `report-writing` skill for structure

## Stage 3 — Final Delivery
- Produce final `.md` document with all feature suggestions
- Deliver via `.docx` using the docx skill

## Constraints
- Every feature must be usable by a 5-year-old (no complex menus, no text-heavy inputs)
- Heavy lifting must be LLM-automated (background processing)
- User sees only magical results on the frontend
- Must respect the existing JSON contract architecture
- Maintain the stamp-based, code-free philosophy
