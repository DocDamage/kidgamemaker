# Plan: Deep Research — Gaming Mechanics for Stamp-Based Zero-Code Game Creator

## Context
- **Project**: Greenfield — stamp-based, zero-code game creation platform for young children (5yo target)
- **Core Philosophy**: Stamp-based placement, zero frontend code, LLM handles all code generation in background
- **Constraint**: A 5-year-old must be able to use it — extreme simplicity in UI, deep complexity in backend features
- **Research Input**: 28 studios spanning 5 decades of side-scrolling innovation, categorized by era

## Stage 1 — Deep Research (deep-research-swarm skill, Route B: Focused Search)

### Phase 1: Landscape Scan
- Macro overview of child-friendly game creation platforms
- Current state of visual/stamp-based game builders (Scratch Jr, GameMaker, etc.)
- LLM-powered code generation in game development tools
- Research each of the 28 studios' key innovations and how they can be simplified for a stamp-based system

### Phase 2: Dimension Decomposition (≥10 dimensions)
1. **Core Platforming Mechanics** — From Mario, Sonic, Celeste: jumping, momentum, coyote time, corner correction
2. **Combat & Action Systems** — From Contra, Metal Slug, Mega Man: shooting, melee, weapon systems
3. **Progression & RPG Layering** — From Castlevania, Wonder Boy, River City Ransom: XP, gear, stats, shops
4. **Traversal & Movement** — From Bionic Commando, Ori, Shantae: grappling, transformations, special movement
5. **World Structure & Level Design** — From Metroid, Dead Cells: interconnected worlds, procedural stitching, backtracking
6. **Puzzle & Temporal Mechanics** — From Braid, Portal-inspired: time manipulation, environmental puzzles
7. **Co-op & Social Mechanics** — From Castle Crashers, Double Dragon: multiplayer, companion AI
8. **Visual & Atmospheric Systems** — From Limbo, Inside, Ori: lighting, diegetic UI, atmosphere
9. **AI & Adaptive Systems** — From Darkest Dungeon, modern AI: enemy behavior, difficulty adaptation, stress/affliction
10. **LLM Integration & Code Generation** — Background code generation, natural language to game logic
11. **Accessibility & Child UX** — Assist modes, stamp simplification, visual feedback for 5yo audience
12. **Roguelike & Procedural Elements** — From Dead Cells, Roguevania: randomization, permanent upgrades, replayability

### Phase 3: Parallel Deep Dive
- Deploy 12 sub-agents in parallel, one per dimension
- Each agent performs ≥20 searches, focusing on the 28 studios' innovations
- Each agent derives child-friendly simplifications and stamp-based adaptations
- Output: dimension files with findings, citations, and actionable recommendations

### Phase 4: Cross-Verification
- Compare all dimension outputs
- Classify confidence tiers
- Surface contradictions
- Save cross-verification results

### Phase 6: Insight Extraction
- Identify cross-dimension patterns
- Extract non-obvious insights
- Formulate feature recommendations

## Stage 2 — Feature Derivation & Code Generation

### Sub-stage 2A: Feature List Compilation
- New features derived from research
- Feature upgrades to existing concepts
- UI/UX improvements for 5yo accessibility

### Sub-stage 2B: Code Snippets
- Real, usable code snippets for each feature
- Edge case handling
- LLM integration patterns
- Stamp-to-code translation layer

## Stage 3 — Report Writing (report-writing skill)
- Compile all research into final deliverable
- Format: Markdown → DOCX
- Include all features, upgrades, UI/UX improvements, code snippets, edge cases
