- **Description**: Spawning external PowerShell processes for unzipping files has been completely removed. The watch daemon now uses the native Rust `zip` crate to open and extract archives in program memory, eliminating shell expansion vulnerabilities.

### ✅ Resolved: Command Injection risk on Workspace Path in `commands.rs`

- **Status**: **RESOLVED**
- **Description**: Spawning PowerShell processes to run `Compress-Archive` during project exports has been replaced with programmatic recursive ZIP compilation using `zip::ZipWriter` and `io::copy`, which runs safely inside the Rust engine runtime context without spawning shell child processes.

### ✅ Resolved: Content Security Policy (CSP) Enabled
- **Location**: [`tauri.conf.json:L22-24`](file:///g:/kidgamemaker/editor/src-tauri/tauri.conf.json#L22-L24)

- **Description**: Disabling Content Security Policy has been replaced with a strict local asset isolation policy:
  `"csp": "default-src 'self' tauri: asset: https://asset.localhost; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: asset: https://asset.localhost blob:;"`

### ✅ Resolved: Zip Slip (Path Traversal) during Ingestion

### ✅ Resolved: Godot Headless WAV Leak with Player + Enemy Scripts
- **Location**: [`Main.gd`](file:///g:/kidgamemaker/engine/scripts/Main.gd), [`PlayerController.gd`](file:///g:/kidgamemaker/engine/scripts/PlayerController.gd), [`SmartEnemy.gd`](file:///g:/kidgamemaker/engine/scripts/SmartEnemy.gd)
- **Status**: **RESOLVED**
- **Root cause**: The enemy damage `Area2D` listened on collision layer 1, while the enemy body itself also used layer 1. On startup the contact area detected its parent enemy body and called `SmartEnemy.take_damage()`, which played hit audio and left Godot headless with leaked WAV playback objects during immediate shutdown.
- **Fix**: `SmartEnemy._on_damage_area_body_entered()` now ignores `self` and only applies contact damage to `Main.active_player`. Audio playback state now lives in `AudioManager.gd`, performs explicit BGM/SFX cleanup, tracks one-shot SFX players, and disables audio playback under the headless display server where there is no audible output and immediate shutdown cannot drain generated playback.
- **Evidence**: Far-apart player+enemy headless probe with audio instrumentation no longer emits startup hit audio, and both the focused probe and `godot --headless --verbose --path engine --quit` complete without `ObjectDB` or `AudioStreamWAV` leak reports.

- **Description**: Sprite animation preview intervals, delayed audio-status timeout callbacks, App beat-preview intervals, and App long-press timers now stop when their UI paths close or the component is destroyed. This prevents hidden modal sessions and hot reloads from leaving timers active.
